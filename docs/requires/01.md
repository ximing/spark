# 产品需求文档（PRD）- AI 实时随身英语助手（macOS）

**版本：** v2.0  
**状态：** 待评审（Draft）  
**作者：** PM & AI 协作  
**日期：** 2026-03-16

---

## 1. 背景与问题陈述

目标用户在日常办公场景（IM 聊天、搜索、文档编辑）中大量使用中文输入，但英语学习常与真实工作语境割裂，导致：

- 英语接触时间短、频率低；
- 学到的表达无法快速迁移到真实沟通；
- "想学英语"与"要完成工作"之间存在切换成本。

本产品希望把用户高频中文输入场景，通过用户主动触发的方式，转化为可参考的英文表达，持续增加英语暴露时间，帮助用户长期沉浸在英语语境中。

---

## 2. 产品目标（V1）

### 2.1 核心目标

围绕用户日常高频对话场景，通过快捷键触发翻译，提升英语接触时长与语境沉浸度。

### 2.2 业务目标

- 用户主动触发翻译，无需担心隐私问题；
- 降低用户从"中文思考"到"英文表达"的心理与操作门槛；
- 验证"快捷键触发翻译"在办公场景中的可行性。

---

## 3. 目标用户与核心场景

### 3.1 目标用户

- 职场白领（高频使用微信/飞书/Slack/浏览器/文档工具输入中文）。

### 3.2 核心场景

- 聊天沟通：输入中文消息时，双击 Control 键翻译整个输入框内容；
- 搜索查询：在搜索框输入中文，双击 Control 键获得英文等价表达；
- 文档写作：在文档输入中文句子，双击 Control 键查看英文版本。

---

## 4. 版本范围定义

根据已确认范围，V1 采用"最小可用版本（MVP）"。

### 4.1 V1 In Scope（必须交付）

1. 全局快捷键触发（双击 Control 键，默认可配置）；
2. 读取当前焦点输入框的全部文本内容；
3. 异步执行翻译，不阻塞用户当前输入；
4. 悬浮窗展示英文翻译结果；
5. 一键复制翻译结果；
6. 大模型设置界面（用户可配置使用哪个模型）；
7. 本地历史记录为可选能力，且默认关闭。

### 4.2 V1 Out of Scope（明确不做）

- 词汇讲解、语法拆解、学习打卡体系；
- 云端历史同步、多端同步；
- 自动改写多风格（口语/商务/学术）高级选项；
- iOS/iPadOS 客户端。

---

## 5. 核心用户流程（User Flow）

1. 用户首次启动应用；
2. 应用引导用户授予 macOS 辅助功能权限（用于全局快捷键和读取输入框内容）；
3. 用户在任意应用输入中文内容；
4. 用户双击 Control 键（或自定义快捷键）触发翻译；
5. 系统读取当前焦点输入框的全部文本；
6. 异步发起翻译请求（不阻塞用户继续输入）；
7. 悬浮窗展示英文结果；
8. 用户可一键复制结果；
9. 用户可在设置页切换模型配置，后续请求按新配置执行。

---

## 6. 功能需求（Functional Requirements）

- **FR-001 权限引导**：系统必须在首次使用时引导并校验辅助功能权限状态。
- **FR-002 全局快捷键**：系统必须在任意应用中使用双击 Control 键触发翻译（可配置自定义快捷键）。
- **FR-003 文本获取**：系统必须读取当前焦点输入框的全部文本内容。
- **FR-004 异步翻译**：翻译请求必须在后台异步执行，不阻塞用户当前输入。
- **FR-005 内容过滤**：系统必须过滤空值、密码框输入，以及本应用自身输入框内容。
- **FR-006 翻译展示**：系统必须通过悬浮窗展示最新英文翻译结果，且不抢占当前输入焦点。
- **FR-007 一键复制**：系统必须支持用户一键复制当前英文翻译结果。
- **FR-008 模型设置界面**：系统必须提供模型配置页，允许用户选择/切换模型。
- **FR-009 模型配置项**：至少包含"模型名称、API Key、Base URL（可选）"并支持连通性测试。
- **FR-010 配置生效**：用户切换模型后，新发起的翻译请求必须使用新配置。
- **FR-011 本地历史（可选）**：系统必须提供"本地历史开关"，默认关闭；开启后仅保存在本机。
- **FR-012 快捷键自定义**：系统必须允许用户在设置中自定义触发快捷键。

## 7. 用户故事与验收标准（User Stories）

### US-001：首次可用引导

**描述**：作为首次使用用户，我希望被清晰引导完成权限授权，以便功能能正常运行。

**Acceptance Criteria：**

- [ ] 首次启动时出现权限引导页；
- [ ] 未授权状态下展示明确提示与"前往系统设置"入口；
- [ ] 授权成功后 3 秒内进入可触发状态。

### US-002：快捷键触发翻译

**描述**：作为职场用户，我希望通过双击 Control 键触发翻译，以便在任意输入场景下学习英文。

**Acceptance Criteria：**

- [ ] 在任意应用中双击 Control 键可触发翻译；
- [ ] 触发后系统读取当前焦点输入框的全部文本；
- [ ] 翻译请求在后台异步执行，不影响用户继续输入；
- [ ] 翻译过程中用户可继续正常操作。

### US-003：悬浮展示与复制

**描述**：作为用户，我希望翻译结果随时可见且可复制，以便快速复用。

**Acceptance Criteria：**

- [ ] 悬浮窗展示最新一条翻译；
- [ ] 悬浮窗不抢焦点，不影响正常输入；
- [ ] 点击复制后写入剪贴板并有成功反馈。

### US-004：模型选择与配置

**描述**：作为用户，我希望自己选择模型，以便在成本、速度、质量之间按需权衡。

**Acceptance Criteria：**

- [ ] 设置页可查看当前生效模型；
- [ ] 用户可新增或编辑模型配置；
- [ ] API Key 以掩码显示并支持更新；
- [ ] "测试连接"可返回成功/失败结果；
- [ ] 切换后新请求使用新模型。

### US-005：本地历史可控

**描述**：作为注重隐私的用户，我希望历史记录默认关闭并可自行开启。

**Acceptance Criteria：**

- [ ] 设置页存在"保存本地历史"开关；
- [ ] 默认状态为关闭；
- [ ] 关闭后不新增历史；开启后仅存本机。

### US-006：快捷键自定义

**描述**：作为用户，我希望自定义触发快捷键以适应个人习惯。

**Acceptance Criteria：**

- [ ] 设置页提供快捷键自定义入口；
- [ ] 支持常见快捷键组合（如 Cmd+Shift+T）；
- [ ] 自定义后立即生效。

## 8. 非功能需求（NFR）

- **NFR-001 性能**：从"触发快捷键"到"翻译展示"目标中位耗时 <= 2.0s（网络正常场景）。
- **NFR-002 稳定性**：翻译服务异常退出后，应用应可正常响应下次触发。
- **NFR-003 可用性**：权限未开启或模型不可用时，需给出可理解、可操作的错误提示。
- **NFR-004 安全性**：API Key 需本地安全存储（系统 Keychain 方案）。
- **NFR-005 隐私性**：默认不保存历史，不上传无关上下文；密码框内容严格忽略；用户主动触发时才读取输入框内容。

---

## 9. 数据与隐私策略

1. 默认不开启历史记录；
2. 历史记录开启后仅本地保存，支持用户手动清空；
3. 密码框、敏感输入场景不采集；
4. 仅为翻译目的发送必要文本到用户配置的大模型服务；
5. 用户主动触发（快捷键）时才读取输入框内容，不做持续监听。

---

## 10. 成功指标（Success Metrics）

当前业务方尚未给出最终量化目标，V1 先定义观测指标并在试运行后确定阈值：

- 日均触发翻译次数（DAU 维度）；
- 用户单日英语暴露时长（可用代理指标：翻译结果可见时长）；
- 复制使用率（复制次数 / 翻译次数）；
- 7 日留存（早期体验用户）。

> 注：本节目标值在首轮内测后补齐，不作为当前文档阻塞项。

## 11. 里程碑与上线标准

### 11.1 里程碑（建议）

- **M1（原型验证）**：打通快捷键触发 -> 读取输入框 -> 翻译 -> 展示最小链路；
- **M2（可用版本）**：补齐复制、错误处理、模型设置；
- **M3（内测版本）**：补齐隐私开关、本地历史可选、快捷键自定义。

### 11.2 上线门槛（V1）

- FR-001 ~ FR-012 全部满足；
- 无 P0/P1 阻塞问题；
- 核心流程（触发、翻译、展示、复制、切换模型）可连续完成。

---

## 12. 风险与应对

- **风险 R1：辅助功能权限获取失败率高**  
  应对：提供分步引导与故障排查提示。

- **风险 R2：部分应用无法读取输入框内容**  
  应对：提供备选方案（如读取剪贴板内容）作为降级策略。

- **风险 R3：模型接口不稳定或成本波动**  
  应对：提供可切换模型与连通性测试，降低单一供应商依赖。

- **风险 R4：隐私敏感导致用户顾虑**  
  应对：默认关闭历史、本地存储优先、明确披露数据路径、用户主动触发才读取。
