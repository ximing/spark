## Codebase Patterns
- Use protocol-based service interfaces for all major features (PermissionService, InputMonitoringService, TranslationService, etc.)
- AppEnvironment acts as a lightweight DI container - update AppEnvironment.production() when replacing mock implementations
- AppState is marked @MainActor and uses @Published properties for SwiftUI binding
- All service protocols are defined in separate files in spark/Services/
- Domain models live in spark/Models/ (PermissionState, TranslationResult, ModelConfig, HistoryItem)
- SwiftUI views should be organized in spark/Views/ directory
- The project uses PBXFileSystemSynchronizedRootGroup - new files in spark/ are automatically included in the build
- Import AppKit when using NSWorkspace or other macOS UI APIs
- AppState.checkPermissions() should be called on init and when returning from System Settings
- Use @EnvironmentObject to inject AppState into SwiftUI views
